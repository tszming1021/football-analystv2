# Project Iron Rules

Last updated: 2026-06-16

## 赛前分析铁律

任何正式赛前分析报告必须严格按完整流程执行。除非用户明确要求“快速版/口播版/只要直白结论”，否则不得跳过关键层，也不得用人工口径冒充完整模型流程。

## 必跑步骤

1. 读取 500 主表数据：三向表、让球三向表、比分表、总球表、半全场表；若来自 PDF/附件，必须说明来源。
2. 读取 500 深层数据：基本面、排名、近况、主客场拆分、交锋、未来赛程、预计阵容、伤停/情报、市场热度。
3. 读取用户附件表格：CSV/XLS/XLSX/DOCX/PDF 必须实际解析；如果扩展名与真实格式不一致，必须记录真实格式和解析方式。
4. 接入 API-Football：fixture、team、league/season、排名/赛程、伤停、首发、技术统计或 prediction；如果 API 无结果，必须写明 endpoint、失败原因和替代来源。
5. 联网复核：伤停、出战、预计首发、发布会、比赛动机、轮换/留力、天气/场地；必须尽量使用官方、足协、球队、权威媒体或明确来源。
6. 运行数学模型：Poisson/Dixon-Coles、概率融合、市场数字融合、让球方向模型、总球模型、比分融合模型。
7. 运行市场数字去水和贝叶斯融合层：正式分析必须调用 `core/market_dewater.py` 将三向表、让球三向表、总球表转为去水隐含概率，再调用 `core/bayesian_fusion.py` 与模型先验融合，输出后验概率、模型/市场偏差、可信度和触发警告。
   - 不得直接使用原始表格数值作为最终概率。
   - 若模型概率与去水市场概率最大偏差超过14%，必须降低信心并写入风险提示；超过22%，必须折扣市场证据或转为保护表达。
   - 让球方向和总球方向必须分别去水，不得只用三向表去水结果替代其他玩法层。
   - 若同时获得Opta模拟、Polymarket或其他外部概率源，赛果层应调用`core/multi_source_fusion.py`一次性融合，禁止用顺序叠加造成次序依赖。
   - 外部来源必须输出基础权重、质量系数、相关性折扣、偏差折扣和最终归一化权重；Polymarket不得与500视为完全独立证据。
   - Opta只有在对阵、发布时间和赛前状态可核验时才允许进入概率层；过期或泛化预测只能作为复核。
   - GPT联网复核、奇门和人工文字判断的直接概率权重必须为0%；新事实应回写数据层后重新计算。
8. 运行 xG/xGA 层：优先从 TheStatsAPI/API-Football 等已有 API 数据源获取真实 xG/xGA；若赛前无真实 xG，则调用 `core/xg_proxy_model.py` 计算 proxy xG/xGA，并在报告中明确真实/估算来源。
   - 国家队近况样本少于5场时，proxy xG/xGA 不得直接使用原始场均进失球，必须按样本可靠度向国家队基准值收缩，并在报告中输出可靠度。
   - 普通国际赛若伤停、首发、战术和换人情报不足，G层必须额外检查低节奏风险，不能只因API近况进球高就抬高总球。
9. 运行总球去偏：不得机械输出 2-3 或 3-4；必须对比独立均值、500均值、最终均值，并说明是否触发去偏。
   - 普通国际赛若触发低节奏保护，必须说明哪些信号导致0-2球权重上调。
   - 强弱差明显且让球较深时，Top1/Top2保持保守，Top3-Top5必须保留3-1、4-0、4-1等后段上沿路径；不得把所有比分压成单一2-0/2-1。
   - 当强弱差达到明显级别（LEG强弱差 >= 1.8、三向强队胜率 >= 60%、或让球深度达到 -1 及以上）时，比分Top3必须至少保留一个高总球/上沿比分；若Top1/Top2为1-0、2-0、1-1等稳态比分，Top3必须承担3-0、3-1、0-3、1-3、4-0、4-1等上沿覆盖任务。
   - 风险比分和冷门比分不得只列0-0、1-1、1-0等低位结果；只要强弱差、定位球、反击、红黄牌风险、体能后程或替补深度支持，冷门比分栏必须同时保留一个低位冷门和一个高总球冷门，例如2-2、1-3、3-2、2-3。
   - 总球数推荐必须显式引用双方实力差距、终结点等级、防线失球样本和让球深度。强弱差越大，越要检查4球及以上路径；不能只因小组首轮、强队控节奏或500总球均值偏低，就把总球全部收缩到2球以内。
   - 决策迭代层不得把原始Top3以外的比分强行拉入Top1/Top2；低节奏保护和后段上沿规则主要作用于总球分布与Top3-Top5风险位，避免比分输出过度保守。
10. 运行 LEG 层：判断强队只有胜面，还是足够支持深度优势；E层必须纳入真实 xG 或 proxy xG 信号；正式详细报告必须输出双方 LEG 修正预期进球、L/E/G 三层10分、综合强弱深度10分和强弱差。
11. 运行复盘校准规则：调用项目已有 calibration JSON，把历史复盘经验转成当前报告的调整提示。
   - 世界杯东道主首战必须拆分两类：若具备高压、快速转换和稳定终结点，Top3-Top5要提高3-0/3-1/4-1等上沿路径；若只是主场热度但终结不稳、核心缺阵或世界杯胜场经验不足，必须提高平局和1-1保护。
   - 弱势方若有明确定位球威胁，或强势方定位球防守不稳，必须单独标记 `set_piece_underdog_threat`，提高弱势方进球和一球边界概率。
   - 世界杯小组首轮若强队胜面高但让球层明显偏保护，必须触发 `worldcup_group_opening_favorite_heat_discount` 或 `worldcup_handicap_over_result_preference`，降低三向强队热度，提高平局和弱势方韧性。
   - 世界杯小组首轮不能机械套用热门降温：若强弱差极大、让球较深、总球4+基础不低或标记 `massive_favorite_depth`，必须触发 `worldcup_massive_favorite_blowout_override`，Top3-Top6保留4-0、5-0、5-1、6-1等连续进球路径。
   - 世界杯小组首轮若连续出现平局潮，必须标记 `group_draw_wave` 并触发 `worldcup_group_draw_wave_guard`；0-0、1-1、2-2必须进入比分保护池，不能只在文字里“防平”。
   - 强队胜面很高但存在轮换、爆点替补、对手低位纪律或门将状态强时，必须标记 `favorite_rotation_risk`、`low_block_opponent` 或 `elite_goalkeeper_risk`，并触发 `favorite_rotation_low_block_depth_discount`；不得直接把强弱差转成深度结论。
   - 强队略优但对手具备组织纪律、快速转换、定位球或追分能力时，必须标记 `organized_transition_underdog`、`set_piece_underdog_threat` 或 `comeback_equalizer_risk`，并触发 `worldcup_transition_2_2_guard`，不能只用1-1保护，2-2也要进入比分保护池。
   - 受让+1场景若存在天气、旅程、低位韧性或弱势方先进球路径，必须标记 `plus_one_cover_risk`、`travel_disruption` 或 `low_block_opponent`，并触发 `plus_one_underdog_cover_guard`，提高让胜保护。
   - 冷门低比分若模型概率接近主线Top3，必须标记 `cold_low_score_near_top` 并触发 `cold_low_score_near_top_promote`；该比分不能只放在冷门栏，必须进入主线保护或在报告中明确列为优先防线。
   - 联赛榜首/强强直接对话若对手防守强或平局权重高，必须标记 `top_table_direct_rival` 或 `strong_defense_opponent`，并检查0-0、1-1是否进入主线保护。
   - 客队方向与市场数字一致且主队防线失球样本偏高时，必须标记 `away_depth_upside` 或 `home_defensive_leak`，Top3-Top5保留0-2、0-3、1-3等客队赢深路径。
   - 开放型联赛或双方失球偏高时，必须标记 `open_league_high_variance` 或提高 `high_scoring_risk`，比分Top4-Top6保留4球以上上沿路径。
12. 运行决策迭代层：调用 `core/decision_iteration.py` 和 `data/calibration/decision_iteration_rules.json`，把复盘经验转成最终推荐前的透明微调；必须输出“调整前 vs 调整后”和触发规则。
   - 决策迭代层只能重排候选比分，不能凭空生成比分；因此在进入决策迭代前，比分候选池必须包含主线Top3、平局保护、冷门低比分、高总球冷门、高波动比分和规则要求的风险比分。
13. 运行一致性检查：检查赛果、让球、总球、比分、LEG、决策迭代层是否互相冲突；冲突必须写入报告。
14. 运行奇门遁甲辅助：只作为低权重风险提示，不得覆盖数据模型。
15. 运行 GPT/AI 二次复核或生成可审计的联网复核块；如果无法调用外部模型，必须说明是 Codex 当前会话联网复核。
16. 按标准模板输出：正式报告优先使用 `report_template_jingcai_qimen.md` 或保持同等章节完整度。
17. 输出前做敏感词扫描和步骤审计；若有步骤未完成，必须在报告首页列出。

## 禁止事项

- 禁止只读部分附件后声称“完整分析”。
- 禁止只用 500 主表直接推出比分和总球。
- 禁止跳过去水和贝叶斯融合，直接把原始市场数字当最终概率。
- 禁止把强队三向优势直接等同于让球方向成立。
- 禁止把普通进失球均值冒充真实 xG；真实 xG 缺失时必须标注为 proxy xG。
- 禁止把奇门、AI复核或人工判断覆盖数学模型概率。
- 禁止隐藏 API、联网、表格解析失败。
- 禁止未跑一致性检查就输出“最终标准报告”。
- 禁止未跑决策迭代层就输出“最终推荐表”。
- 禁止敏感词进入用户-facing 正式报告正文。

## 快速版例外

快速版、直白版、口播版可以省略部分计算，但必须在文件或回答中标注：

```text
这是快速版，不等同于标准完整报告。
```

快速版不得替代标准完整报告存档。
